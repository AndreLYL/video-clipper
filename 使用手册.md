## 📝 软件简介

VideoClipper 是一款专为视频片段裁剪设计的桌面应用程序，采用现代化深色主题UI设计，操作简单直观。软件支持单点裁剪和批量裁剪两种模式，能够快速、精确地从长视频中提取指定时间段的片段。

### 核心优势

- ✅ **无需安装**：单文件EXE，双击即用
- ✅ **操作简单**：图形化界面，无需专业知识
- ✅ **功能强大**：支持单点和批量裁剪
- ✅ **智能识别**：自动修复无法读取的视频
- ✅ **精美报告**：批量裁剪自动生成HTML报告
- ✅ **多格式支持**：支持多种时间格式输入

## 📖 详细功能说明

### 一、单点裁剪模式

#### 功能说明

单点裁剪模式用于从视频中提取单个特定时间段的片段。您需要提供：
- 源视频文件
- 视频起始UTC时间
- 要裁剪的时间点
- 向前/向后裁剪秒数

#### 工作原理

假设：
- 视频起始时间：`12:00:00`
- 裁剪时间点：`12:30:45`
- 向前40秒，向后20秒

程序会计算：
- 裁剪起始点：`12:30:45 - 40秒 = 12:30:05`
- 裁剪结束点：`12:30:45 + 20秒 = 12:31:05`
- 最终输出：60秒的视频片段（从12:30:05到12:31:05）

#### 输出文件命名

- 格式：`HH-MM-SS.mp4`
- 示例：`12-30-45.mp4`

### 二、批量裁剪模式

#### 功能说明

批量裁剪模式可以从文本文件中读取多个时间点，一次性完成所有片段的裁剪。特别适合：
- 从长视频中提取多个精彩片段
- 批量处理监控录像
- 提取会议记录中的关键片段

#### 时间戳文件格式

创建一个文本文件（如 `timestamps.txt`），每行一个时间点。

**基本格式**：
```
时间点 描述（可选）
```

**支持的格式示例**：
```
# 这是注释行，以#开头
# 空行会被忽略

# 格式1: HH:MM:SS
12:30:45 第一个片段

# 格式2: HH:MM（自动补全秒数）
12:37 第二个片段

# 格式3: YYYY-MM-DD HH:MM:SS
2025-11-13 00:26:39 第三个片段

# 格式4: YYYY年MM月DD日HH:MM:SS
2025年11月13日00:26:50 第四个片段

# 格式5: HH点MM分SS秒
00点34分20秒 第五个片段

# 格式6: HH点MM分（自动补全秒数）
12点37分 第六个片段
```

**注意事项**：
- 以 `#` 开头的行为注释，会被忽略
- 空行会被忽略
- 时间和描述之间可以有空格，也可以没有
- 支持中文标点符号（如：、－等），会自动转换
- 如果缺少秒数，会自动使用"向前裁剪秒数"作为默认值

#### 输出文件命名

- 格式：`HH-MM-SS_描述.mp4`
- 示例：`12-30-45_第一个片段.mp4`
- 如果没有描述，则为：`12-30-45.mp4`

#### HTML报告

批量裁剪完成后，程序会自动生成HTML报告，包含：

1. **统计信息**
   - 总计片段数
   - 成功数量
   - 失败数量

2. **每个片段的详细信息**
   - 时间点
   - 描述（如果有）
   - 输出文件名
   - 状态（成功/失败）
   - 错误信息（如果失败）

3. **视频帧预览**
   - 每个成功片段的首帧和尾帧
   - 点击图片可放大查看
   - 支持ESC键或点击背景关闭

4. **报告位置**
   - 保存在视频输出目录
   - 文件名：`裁剪报告_YYYYMMDD_HHMMSS.html`

## 📋 操作步骤详解

### 场景一：单点裁剪

#### 准备工作

1. **准备源视频**
   - 确保视频文件完整，可以正常播放
   - 支持的格式：MP4、AVI、MOV、MKV、FLV等

2. **确定时间信息**
   - 视频起始UTC时间（例如：`12:00:00`）
   - 要裁剪的时间点（例如：`12:30:45`）
   - 裁剪时长（向前40秒，向后20秒）

#### 操作步骤

**步骤1：启动程序**
- 双击 `VideoClipper_v1.3.2.exe`
- 等待程序加载完成

**步骤2：选择源视频**
- 点击"源视频"右侧的"浏览"按钮
- 在文件选择对话框中选择视频文件
- 确认文件路径显示在输入框中

**步骤3：选择保存目录**
- 点击"保存目录"右侧的"浏览"按钮
- 选择要保存裁剪视频的文件夹
- 可以创建新文件夹
- 确认目录路径显示在输入框中

**步骤4：选择裁剪模式**
- 点击"单点裁剪"单选按钮
- 确认"裁剪时间点"输入框显示

**步骤5：配置视频起始时间**
- 在"视频起始时间 (UTC)"输入框中输入
- 格式：`HH:MM:SS`
- 示例：`12:00:00`

**步骤6：配置裁剪时长（可选）**
- "向前"输入框：默认40秒（可修改）
- "向后"输入框：默认20秒（可修改）
- 这两个值决定了裁剪片段的长度

**步骤7：输入裁剪时间点**
- 在"裁剪时间点"输入框中输入
- 格式：`HH:MM:SS`
- 示例：`12:30:45`
- 注意：时间点不能早于视频起始时间

**步骤8：开始裁剪**
- 点击"▶ 开始裁剪"按钮
- 观察进度条和状态信息
- 等待处理完成

**步骤9：查看结果**
- 处理完成后会弹出成功提示
- 在保存目录中找到输出的视频文件
- 文件名格式：`HH-MM-SS.mp4`

#### 示例演示

假设您有一个从 `12:00:00` 开始的视频，想要提取 `12:30:45` 这个时间点前后各40秒和20秒的片段：

1. 源视频：`D:\Videos\source.mp4`
2. 保存目录：`D:\Videos\clips`
3. 视频起始时间：`12:00:00`
4. 向前秒数：`40`
5. 向后秒数：`20`
6. 裁剪时间点：`12:30:45`

点击"开始裁剪"后，程序会：
- 计算裁剪范围：`12:30:05` 到 `12:31:05`（共60秒）
- 生成文件：`D:\Videos\clips\12-30-45.mp4`

### 场景二：批量裁剪

#### 准备工作

1. **准备源视频**（同单点裁剪）

2. **准备时间戳文件**

2025-11-13 00:26:50 异常行为2

# 下午的片段

**步骤1-4：同单点裁剪**
- 启动程序
- 选择源视频
- 选择保存目录
- 选择"批量裁剪"模式

**步骤5：配置视频起始时间**
- 输入视频起始UTC时间
- 格式：`HH:MM:SS`

**步骤6：配置裁剪时长（可选）**

**步骤7：选择时间戳文件**
- 点击"时间戳文件"右侧的"选择文件"按钮

**步骤9：查看结果**
- 处理完成后会显示统计信息

- 报告包含所有片段的详细信息
- 可以点击图片放大查看首帧和尾帧

**时间戳文件内容**：

- 每个时间点生成一个60秒的视频片段
- 文件名包含时间戳和描述

**输出结果**：
- `2025-11-13_00-26-39_人员进入.mp4`
- `2025-11-13_00-26-50_可疑动作.mp4`
- `00-34-20_物品移动.mp4`

## 🎯 高级技巧

### 技巧1：灵活使用时间格式

批量裁剪支持多种时间格式，您可以根据数据来源选择最方便的格式：

- **从日志文件复制**：使用 `YYYY-MM-DD HH:MM:SS` 格式
- **手动记录**：使用 `HH:MM:SS` 或 `HH点MM分SS秒`
- **混合使用**：同一文件中可以混合使用不同格式

### 技巧2：利用注释组织时间戳

在时间戳文件中使用注释可以帮助组织内容：

```
# 第一组片段 - 上午
12:30:45 片段1
12:35:20 片段2

# 第二组片段 - 下午
14:15:30 片段3
14:20:10 片段4
```

### 技巧3：调整裁剪时长

根据实际需求调整向前/向后秒数：

- **短片段**：向前20秒，向后10秒（共30秒）
- **标准片段**：向前40秒，向后20秒（共60秒，默认）
- **长片段**：向前60秒，向后30秒（共90秒）

### 技巧4：使用描述信息

在时间戳文件中添加描述，输出文件名会更清晰：

```
12:30:45 精彩进球
12:35:20 犯规回放
14:15:30 赛后采访
```

输出文件：
- `12-30-45_精彩进球.mp4`
- `12-35-20_犯规回放.mp4`
- `14-15-30_赛后采访.mp4`

### 技巧5：查看HTML报告

HTML报告不仅显示统计信息，还包含：
- 每个片段的首帧和尾帧预览
- 点击图片可以全屏查看
- 失败的项目会显示详细错误信息
- 可以保存报告文件，方便后续查看

### 技巧6：处理大文件

对于大视频文件：
- 确保有足够的磁盘空间（输出文件需要空间）
- 批量裁剪时，处理时间可能较长，请耐心等待
- 可以分批处理，将时间戳文件分成多个小文件

### 技巧7：错误处理

如果某些时间点裁剪失败：
- 查看错误信息，了解失败原因
- 常见原因：
  - 时间点超出视频范围
  - 时间点太早（需要的前置秒数不足）
  - 时间点太晚（需要的后置秒数不足）
- 修正时间戳文件后重新处理

## ❓ 常见问题解答

### Q1: 程序无法启动怎么办？

**A:** 可能的原因和解决方法：
1. **被杀毒软件拦截**
   - 将程序添加到杀毒软件的白名单
   - 或者暂时关闭杀毒软件

2. **系统不兼容**
   - 确保系统是Windows 10或11
   - 尝试以管理员身份运行

3. **首次运行慢**
   - 首次运行需要10-30秒加载，这是正常现象
   - 请耐心等待

### Q2: 裁剪失败，提示"时间超出范围"？

**A:** 可能的原因：
1. **裁剪时间点早于视频起始时间**
   - 检查"视频起始时间"设置是否正确
   - 确保裁剪时间点不早于起始时间

2. **需要的前置秒数不足**
   - 如果时间点太早，无法向前裁剪足够的秒数
   - 建议：选择更晚的时间点，或减少"向前秒数"

3. **需要的后置秒数不足**
   - 如果时间点太晚，无法向后裁剪足够的秒数
   - 建议：选择更早的时间点，或减少"向后秒数"

4. **视频时长不足**
   - 检查视频文件是否完整
   - 确认视频时长是否足够

### Q3: 批量裁剪部分失败怎么办？

**A:** 处理方式：
1. **查看错误信息**
   - 程序会显示失败的项目和原因
   - HTML报告中也包含详细的错误信息

2. **修正时间戳文件**
   - 根据错误信息修正时间点
   - 重新运行批量裁剪

3. **继续处理**
   - 程序会跳过失败的项目，继续处理其他时间点
   - 成功的片段会正常保存

### Q4: 时间格式输入错误怎么办？

**A:** 格式要求：
1. **单点裁剪模式**
   - 必须使用 `HH:MM:SS` 格式
   - 例如：`12:30:45`（不能是 `12:30`）

2. **批量裁剪模式**
   - 支持多种格式（见上文）
   - 如果格式不正确，程序会提示错误
   - 检查时间戳文件，确保格式正确

### Q5: 输出的视频文件在哪里？

**A:** 文件位置：
- 输出文件保存在您选择的"保存目录"中
- 单点裁剪：直接保存在该目录
- 批量裁剪：所有片段保存在该目录，HTML报告也在该目录

### Q6: 可以修改输出视频的质量吗？

**A:** 当前版本：
- 输出视频使用标准质量设置
- 未来版本可能会添加质量选项

### Q7: 支持哪些视频格式？

**A:** 支持的格式：
- MP4（推荐）
- AVI
- MOV
- MKV
- FLV
- 其他常见视频格式

### Q8: 程序会自动修复无法读取的视频吗？

**A:** v1.3.2新特性：
- 是的，程序会自动检测并修复无法读取的视频
- 使用ffmpeg自动重封装
- 需要系统已安装ffmpeg并加入PATH
- 如果修复失败，会显示详细错误信息

### Q9: HTML报告可以分享吗？

**A:** 可以：
- HTML报告是独立的文件
- 包含所有图片（base64编码）
- 可以复制到其他电脑查看
- 在浏览器中打开即可

### Q10: 如何卸载程序？

**A:** 卸载方法：
- 直接删除EXE文件即可
- 程序不会在系统中留下任何痕迹
- 不会修改注册表或系统文件

## 🎬 视频制作脚本

### 第二部分：单点裁剪演示（3-4分钟）

#### 场景设置
- 准备一个示例视频文件
- 确定要裁剪的时间点

#### 操作步骤（详细演示）

**步骤1：选择源视频（30秒）**
- 画面：点击"浏览"按钮
- 旁白："首先，我们需要选择源视频文件。点击源视频右侧的浏览按钮。"
- 画面：在文件选择对话框中选择视频
- 旁白："在文件选择对话框中选择您的视频文件，支持的格式包括MP4、AVI、MOV等常见格式。"
- 画面：确认文件路径显示在输入框

**步骤2：选择保存目录（20秒）**
- 画面：点击保存目录的"浏览"按钮
- 旁白："接下来选择保存目录，这是裁剪后视频的保存位置。"
- 画面：选择或创建文件夹
- 旁白："您可以选择现有文件夹，也可以创建新文件夹。"

**步骤3：选择单点裁剪模式（10秒）**
- 画面：点击"单点裁剪"单选按钮
- 旁白："我们选择单点裁剪模式，用于提取单个视频片段。"

**步骤4：配置参数（1分钟）**
- 画面：输入视频起始时间
- 旁白："输入视频的起始UTC时间，格式为HH:MM:SS，例如12:00:00。这个时间表示视频开始录制的时间。"
- 画面：显示向前和向后秒数输入框
- 旁白："这里可以配置裁剪时长。向前秒数表示从时间点向前裁剪多少秒，向后秒数表示向后裁剪多少秒。默认是向前40秒，向后20秒，总共60秒的片段。"
- 画面：输入裁剪时间点
- 旁白："最后输入要裁剪的时间点，格式同样是HH:MM:SS。这个时间点不能早于视频起始时间。"

**步骤5：开始裁剪（1分钟）**
- 画面：点击"开始裁剪"按钮
- 旁白："配置完成后，点击开始裁剪按钮。"
- 画面：显示进度条和状态信息
- 旁白："程序会显示处理进度。处理时间取决于视频大小和片段长度。"
- 画面：显示成功提示
- 旁白："处理完成后会弹出成功提示，显示输出文件的路径。"

**步骤6：查看结果（30秒）**
- 画面：打开保存目录
- 旁白："让我们打开保存目录查看结果。"
- 画面：显示输出的视频文件
- 旁白："可以看到，输出文件以时间戳命名，格式为HH-MM-SS.mp4。这就是我们裁剪的视频片段。"
